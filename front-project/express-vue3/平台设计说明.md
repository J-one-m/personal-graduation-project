# 高校运动场地智能化服务平台


## 参与者

1. 普通用户
    - 行为：预约场地、查看场地使用情况、参与活动以及提供反馈。

2. 场地维护管理员
    - 负责场地的日常管理和维护、处理用户的预约和反馈，确保场地的良好运作


3. 系统管理员
    - 活动的发布取消
    - 用户管理（例如注册、权限分配等）。
    - 数据管理和系统安全。
    - 处理技术问题和系统更新


## 主要功能

1. 场地预约系统:
- 用户可以浏览可用场地，实时查看空闲信息，进行在线预约。
- 支持预约修改和取消功能。

2. 场地管理:
- 场地维护管理员可以添加、编辑和删除场地信息，管理场地的使用状态和维护记录。

3. 活动发布与管理:
- 系统管理员可以创建、发布和取消体育活动，管理活动的参与者信息，确保活动的顺利进行。

4. 通知与提醒:
- 系统自动向用户发送预约确认、活动提醒和重要通知。

5. 数据统计与分析:
- 提供使用情况的统计报告，帮助管理者评估场地使用效率和用户需求。
- 使用echarts，使场地使用情况实时可视化。

6. 用户反馈系统:
- 用户可以对场地和活动进行评价，提供反馈意见，帮助改进服务。
- 私信功能（用户之间），评价。
- 可以发布失物招领

7. 个人信息管理:
- 用户可以查看和管理自己的个人信息（用户名、账号信息、头像管理）、预约记录和活动参与情况。

### 候补表

```sql

-- 候补队列表
CREATE TABLE waitlist (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '候补记录ID',
    
    -- 核心关联字段
    user_id INT NOT NULL COMMENT '申请候补的用户ID',
    time_slot_id BIGINT NOT NULL COMMENT '关联的时段ID',
    venue_id INT NOT NULL COMMENT '关联的场地ID（冗余存储，方便查询）',
    schedule_date DATE NOT NULL COMMENT '关联的日期（冗余存储，方便查询）',
    
    -- 状态管理
    status ENUM('pending', 'notified', 'confirmed', 'expired', 'cancelled') 
        NOT NULL DEFAULT 'pending' COMMENT '候补状态',
    
    -- 优先级计算（核心智能体现）
    priority_score INT NOT NULL DEFAULT 0 COMMENT '优先级分数',
    
    -- 时间字段
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- 外键约束
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (time_slot_id) REFERENCES time_slots(id) ON DELETE CASCADE,
    FOREIGN KEY (venue_id) REFERENCES venues(id),
    
    -- 索引优化
    UNIQUE KEY uk_user_time_slot (user_id, time_slot_id, status) COMMENT '防止用户重复申请同一时段',
    INDEX idx_priority (time_slot_id, status, priority_score DESC, created_at ASC) COMMENT '按优先级排序',
    INDEX idx_status_time (status, created_at) COMMENT '状态和时间查询'
) COMMENT='场地预约候补队列表';

```

==在数据库设计中，“几乎所有的冗余都是为了性能或业务的清晰性而存在的”。max_waitlist正是这样一个为了业务清晰和并发安全而存在的、有价值的“冗余”字段。==

### 智能化体现：

1. 数据驱动的用户信用体系：
    - 智能化目标：用数据公平的奖励守信用户、约束失信行为、建立系统内的正向激励机制。
    - 具体规则：基于用户的准时签到率（准时签到次数 / 总签到次数）自动计算信用分（可能会使用自己的实现逻辑：总分10分、一次未到场扣两分，超时5分钟扣一分（15分钟后还未到场的原本系统会自动将其状态更新为未到场。）、低于5分限制其预约场地），并根据信用分自动授予或限制权限。
    - 信用分的应用：对于高信用用户：可延长到场时长5~10分钟；对于低信用用户：严格限制其到场时间。

2. 动态资源调度与候补自动化：
    - 智能化目标：解决“占着茅坑不拉屎”的资源浪费问题，实现资源利用率最大化
    - 具体规则：若用户在预约开始后的宽限期时间内（因用户当前信用而定）未扫码签到，那么系统自动将预约标记为“未到场”，并立即自动触发通知，将该时段释放给候补队列（候补队列最大两位）中的用户。
    - 执行方式：系统自动发送邮件或者信息通知候补的用户在规定时间内到场。
